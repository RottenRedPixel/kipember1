# Ember Screenshot Plan

Last updated: March 21, 2026

Apple’s current App Store Connect screenshot specs accept multiple iPhone sizes. For a minimal iPhone-only submission, capture one consistent 6.9-inch device set if possible.

## Recommended device family

- Preferred: any accepted 6.9-inch iPhone screenshot size
  - `1320 x 2868`
  - `1290 x 2796`
  - `1260 x 2736`
- Fallback if you do not provide 6.9-inch shots: `1242 x 2688` or `1284 x 2778` for 6.5-inch

## Shot list

1. Feed / archive overview
   - Show multiple memories in the archive
   - Caption idea: `Every memory stays connected`
2. Upload / create flow
   - Show the start of creating a new Ember
   - Caption idea: `Start with a photo, video, or story`
3. Memory detail / wiki view
   - Show one memory with generated context and summary
   - Caption idea: `Turn one photo into a living record`
4. Story Circle / contributors
   - Show how friends or family can add their perspective
   - Caption idea: `Invite the people who lived it`
5. Voice or chat memory flow
   - Show text or voice contribution in context
   - Caption idea: `Capture real voices and reflections`

## Capture rules

- Use real-looking sample memories, not blank states
- Keep status bars consistent across the full set
- Avoid debug banners, dev menus, or localhost URLs
- Use the same device size for all screenshots in one localization
- Prefer portrait for the first release unless landscape is a core experience

## Submission note

Apple currently requires at least one and allows up to ten screenshots per device size.
